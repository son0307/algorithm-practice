## 문제 요약

- 어떤 숫자에서 k개의 숫자를 제거하여 얻을 수 있는 가장 큰 수를 반환하는 문제
    - number는 2자리 이상, 1,000,000자리 이하의 자연수

## 핵심 포인트

이 문제의 입력인 number의 자릿수는 최대 1,000,000이다. 입력값의 길이가 매우 길기 때문에 해당 문제는 $O(N^2)$의 시간복잡도를 가지는 알고리즘으로 풀 수 없고  $O(N)$또는 $O(NlogN)$으로 해결해야 한다는 것을 짐작할 수 있다.

큰 수를 만들기 위해선 높은 자릿수에 큰 수가 들어가도록 해야한다. 그러기 위해선 앞에서부터 숫자들을 살펴보며 작은 숫자들을 걸러내고 가장 큰 수를 맨 앞에 세워야 한다. 이 때 사용될 수 있는 것이 스택이다. 숫자들을 앞에서부터 하나씩 살펴보며 지금 넣으려는 숫자가 스택의 가장 위에 있는 수보다 크면 이보다 작은 이전 숫자들을 제거하는 방식으로 구현할 수 있다.

동작 원리는 다음과 같다.

- number의 숫자를 하나씩 스택에 담는다
- 스택에 숫자가 들어있고, 숫자를 더 제거할 수 있으면 스택의 가장 위에 있는 수와 현재 숫자를 비교한다.
- 현재 숫자 > 스택의 가장 위에 있는 숫자이면, 가장 위에 있는 숫자를 제거하고, 제거할 숫자 카운트(k)를 1줄인다. (조건을 만족하는 한 반복)
- 현재 숫자를 스택에 넣는다.
- 모든 숫자를 확인했는데 제거할 숫자 카운트가 0이 아니라면 뒤에서부터 남은만큼 잘라낸다.

### 전체 코드

```java
class Solution {
    public String solution(String number, int k) {
        StringBuilder sb = new StringBuilder();
        int len = number.length();

        for (int i = 0; i < len; i++) {
            char c = number.charAt(i);

            while (k > 0 && sb.length() > 0 && sb.charAt(sb.length() - 1) < c) {
                sb.deleteCharAt(sb.length() - 1);
                k--;
            }
            sb.append(c);
        }

        if (k > 0) {
            sb.setLength(sb.length() - k);
        }

        return sb.toString();
    }
}
```

위 알고리즘의 시간복잡도는 $O(N)$이다. (N은 number의 길이). number 문자열에 담긴 문자들을 앞에서부터 하나씩 차례대로 검사하기 때문이다. 별도로 Stack 자료구조를 선언하여 사용할 수 있지만 여기서는 StringBuilder 자체를 스택으로 생각하고 문제를 해결하였다.
