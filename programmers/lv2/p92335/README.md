## 문제 요약

- 양의 정수 n을 k진수로 변환했을 때, 특정 조건(양 옆이 0이거나 수의 끝부분인 경우)을 만족하는 소수의 개수를 구하는 문제
    - 변환된 수 안에서 찾은 소수는 0을 포함하지 않아야 함
    - 소수 판별은 10진법으로 보았을 때의 값을 기준으로 함
    - 중복되는 소수도 개별적으로 카운트

## 핵심 포인트

문제에서 제시한 조건(0P0, P0, 0P, P)를 개별적으로 구현하려고 하면 로직이 복잡해질 수 있다. 하지만, 해당 조건은 **0을 구분자로 하여 숫자를 분리**하면 모든 조건이 자동적으로 만족된다는 점이 핵심이다.

1. **진법 변환 및 분할** : 주어진 정수 `n` 을 `k` 진수로 변환하고 0을 기준으로 분할하여 검사 대상들을 추출한다.
2. **데이터 타입 주의** : 정수 `n` 의 최댓값은 1,000,000으로 이를 가장 작은 진수인 3으로 변환하면 1,212,210,202,001으로 int 타입의 범위를 넘는다. 따라서, 변환된 수는 long 타입으로 관리해야 한다.
3. **소수 판별** : **$O(\sqrt{N})$** 효율성을 가진 알고리즘을 이용하여 시간 초과를 방지한다.

동작 원리는 다음과 같다.

1. **진법 변환** : 정수 `n`을 `k` 진법 문자열로 변환한다.
2. **검사 대상 추출** : 문자열을 ‘0’ 기준으로 자른다. 이 때, ‘00’과 같이 0이 연속된 경우 빈 문자열이 (””) 나올 수 있으므로 이에 대한 예외 처리가 필요하다.
3. **소수 검사 및 카운트**
    1. 추출된 각 문자열을 10진수(long)으로 변환한다.
    2. 변환된 문자열에 대해 소수 검사를 수행한다. (2 미만인 경우 제외, 2부터 해당 수의 제곱근까지 나누어떨어지는지 확인)

   > 모든 약수는 짝이 존재한다. 즉, 제곱근을 기준으로 작은 수들 중 약수가 존재하면 이에 짝이 되는 약수가 큰 수들 중 존재한다. 따라서, 작은 수들만 검사하여 약수가 존재하지 않는 것이 확인되면 해당 수는 소수이다.
>
4. **결과 반환** : 소수로 판명된 숫자들의 수를 반환한다.

### 전체 코드

```java
class Solution {
    public boolean isPrime(long n) {
        if (n < 2) return false;

        for (long i = 2; i * i <= n; i++) {
            if (n % i == 0) {
                return false;
            }
        }
        return true;
    }

    public int solution(int n, int k) {
        int answer = 0;
        String[] nums = Integer.toString(n,k).split("0");

        for (String num : nums) {
            if (num.isEmpty()) continue;

            if (isPrime(Long.parseLong(num))) {
                answer++;
            }
        }

        return answer;
    }
}
```

위 알고리즘의 시간 복잡도는 다음과 같다.

- 진법 변환 및 문자열 분리: $O(\log_k n)$
- 소수 판별: 분리된 각 숫자의 개수를 M, 각 숫자의 크기를 V라고 하면, $O(M \times \sqrt{V})$

결과적으로, $O(\log_k n + M\sqrt{V})$이 된다.
