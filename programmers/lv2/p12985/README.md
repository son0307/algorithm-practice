## 핵심 포인트

이 문제는 토너먼트라는 포맷을 트리로 이해하는 것이 중요하다. n명이 남아 있다고 할 때, n/2명은 왼쪽 서브 트리에, 나머지는 오른쪽 서브 트리에 있다고 생각하면 된다. 이 때, 우리가 찾아야하는 a와 b가 서로 다른 서브 트리에 존재하는 경우 이 둘이 만나려면 트리의 루트 노드까지 올라와야 하는 것이다.

그림을 통해 이 말의 의미를 좀 더 살펴보도록 하자.

![image.png](attachment:34c7e5a0-b5d5-4142-b8e8-89bcd67f6b75:6967ea89-e29d-43ea-968b-2f468491b653.png)

다음과 같이 트리로 나타낼 수 있다. 2번과 4번이 왼쪽 서브 트리에 같이 존재하므로 현재로서는 이 둘이 언제 만나는 지를 확실히 알 수 없다. 따라서 탐색 범위를 왼쪽 서브 트리로 한정하여 탐색을 진행해야 한다.

![image.png](attachment:20d6790f-9bd5-4478-b713-d612f2be7d5c:baa962ed-6bc0-4b87-8d7e-f736c2cd35a1.png)

왼쪽 트리로 한정하여 동일하게 서브 트리를 검사한다. 여기서는 2번과 4번이 각각 다른 서브 트리에 존재하므로 이 둘이 만나려면 현재 트리의 루트 노드까지 올라와야 한다. 따라서, 이 둘이 만나는 라운드는 트리의 높이, 2 라운드가 된다.

### 전체 코드

```java
public int solution(int n, int a, int b)
{
    // a, b sort
    if(b < a) {
        int temp = a;
        a = b;
        b = temp;
    }

    while (!(a<=n/2 && b>n/2)) {
        if (a > n/2) {
            a -= n/2;
            b -= n/2;
        }

        n/=2;
    }

    return (int)(Math.log(n)/Math.log(2));
}
```
