### 문제 요약

- 초 단위로 주식 가격이 주어졌을 때, 가격이 떨어지지 않은 기간 (초 단위)를 계산
    - 예제를 분석하면 다음과 같이 나온다.
    - 1초 → 1원, 끝날때까지 이보다 낮아지지 않았으므로 4초
    - 2초 → 2원, 끝날때까지 이보다 낮아지지 않았으므로 3초
    - 3초 → 3원, 4초 때 이보다 낮아졌으므로 1초
    - 4초 → 2원, 끝날때까지 이보다 낮아지지 않았으므로 1초
    - 5초 → 3원, 마지막은 항상 0초

## 핵심 포인트

문제를 단순히 풀어보면 매 초를 살펴볼 때마다 앞에서 입력된 price들을 살펴보고 새로 입력된 가격보다 높은 가격을 가진 곳의 위치에 가격이 떨어지지 않은 시간을 계산하여 저장하면 된다.

하지만, 이 방법의 시간복잡도는 O(N^2)으로, 최대 prices의 길이가 100,000이라 최대 연산 횟수가 100억번이 되어 시간 초과가 발생한다. 따라서, 다른 방식의 해결법을 찾아야 한다.

이 문제는 입력된 원소들의 짝을 찾는 문제로 생각할 수 있다. 입력된 가격 이후로 더 작은 가격과의 짝을 맞춘다고 생각하면 되고, 스택을 이용하여 해결할 수 있다.

스택을 이용할 때 생각해야 하는 조건에 대해 먼저 생각해보도록 하자.

- 스택에 저장할 값
    - 가격이 떨어지지 않은 기간을 배열 기준으로 생각한다면 인덱스의 차로 볼 수 있다. 그러므로, 짝이 이뤄진 원소들의 인덱스의 차를 구해야 하므로 스택에는 인덱스를 저장하도록 하였다.
- 더 큰 값이 입력된 경우
    - 입력된 값이 다른 값들보다 짝을 찾는데 우선순위를 가지게 된다. 그러므로 이를 스택에 넣는다.
- 더 작은 값이 입력된 경우
    - 입력된 값보다 작거나 같은 값이 나올때까지 스택에서 값을 빼고 인덱스의 차를 계산하는 것을 반복
- 같은 값이 입력된 경우
    - 단순히 스택에 추가하여 짝을 찾도록 한다.

이와 같은 방식으로 문제를 해결하면 O(N)의 복잡도로 문제를 해결할 수 있다.

### 전체 코드

```java
import java.util.Stack;

class Solution {
    public int[] solution(int[] prices) {
        int[] answer = new int[prices.length];
        Stack<Integer> st = new Stack<>();

        for (int i = 0; i < prices.length; i++) {
            while (!st.empty() && prices[st.peek()] > prices[i]) {
                int index = st.pop();
                answer[index] = i - index;
            }

            st.push(i);
        }

        while(!st.empty()) {
            int index = st.pop();
            answer[index] = (answer.length - 1) - index;
        }

        return answer;
    }
}
```
