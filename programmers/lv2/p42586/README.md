### 문제 요약

- 각 기능의 개발도가 100 이상을 달성하고 난 뒤, 배포 시 동시에 몇개씩 배포되는 지 계산하는 문제
    - 단, 100을 달성했다고 바로 배포되는 것이 아닌, 앞에 있는 기능들 중 100을 달성하지 못한 기능이 있으면 배포 되지 않음.
    - 해당 기능도 개발도 100 이상을 달성해야 그 뒤에 있는 기능들도 배포가 가능.
    - 배포 시 앞에서부터 순차적으로 100 이상인 기능들이 동시에 배포됨.

## 핵심 포인트

전달된 progresses 배열에서 앞에 있는 기능이 개발이 완료되어야 뒤에 있는 기능들도 배포할 수 있기에 앞에 있는 기능이 배포 시 우선권을 가진다. 즉, 뒤에 있는 개발이 완료된 기능들의 배포 시점은 그들의 앞에 있는 기능들 중 가장 늦은 배포 시점을 가진 기능과 동기화되는 것이다.

앞에 있는 것들을 우선적으로 고려해야 하는 것을 생각하면 이 문제에서는 큐를 활용할 수 있다. 첫번째 예제를 설명하면 다음과 같다.

- 1번째 기능 :  완성도 93, 매일 1씩 완성 → 7일 소요
- 2번째 기능 : 완성도 30, 매일 30씩 완성 → 3일 소요
- 3번째 기능 : 완성도 55, 매일 5씩 완성 → 9일 소요

각 기능이 완성되는데 위와 같이 소요되고, 각 배포시 배포되는 기능의 수는 다음과 같다.

- 첫 번째 배포 → 7일차, 1번째 기능과 2번째 기능 배포
- 두 번째 배포 → 9일차, 3번째 기능 배포

이를 통해 도출해낼 수 있는 결론은 **각 기능의 개발 소요 시간을 기준으로 맨 앞 기능의 개발 완료 시점보다 더 적은 개발 완료 시점을 가진 기능들은 같이 배포된다.** 이다. 이를 큐에 빗대어 생각하면 다음과 같이 처리 순서를 결정할 수 있다.

1. 큐에서 맨 앞에 있는 원소 (개발 소요 시간)을 뽑는다.
2. 해당 원소보다 더 큰 원소가 뽑힐 때까지 계속해서 뽑는다.
3. 뽑은 원소의 수를 카운트하여 배열에 저장한다. 이 값이 한 번 배포할 때 배포되는 기능의 수이다.

### 전체 코드

```java
public int[] solution(int[] progresses, int[] speeds) {
      List<Integer> answer = new ArrayList<>();
      Queue<Integer> q = new LinkedList<>();

      for (int i = 0; i < progresses.length; i++) {
          int day = (int) Math.ceil((double) (100 - progresses[i]) / speeds[i]);

          q.add(day);
      }
```

우선, 각 기능이 완전히 개발될 때까지의 기간을 계산하여 이들을 큐에 순차적으로 넣는다.

```java
	while(!q.isEmpty()) {
        int day = q.poll();
        int count = 1;
        while (!q.isEmpty() && q.peek() <= day) {
            count++;
            q.poll();
        }
        answer.add(count);
    }

    return answer.stream().mapToInt(i -> i).toArray();
}
```

이후, 앞에서 설명한 것과 같이 맨 앞의 원소를 하나 빼고, 이 원소보다 큰 값을 만날 때까지 큐에서 원소를 계속해서 빼낸다. 그와 동시에 빼낸 원소의 개수를 카운트한다. 큐가 비거나 더 큰 값을 만나 반복문이 종료되면, 카운트한 값을 정답에 추가한다.
