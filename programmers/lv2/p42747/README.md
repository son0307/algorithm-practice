## 핵심 포인트

이 문제는 단순히 1부터 최대치까지 전부 탐색하면 정답이 나오기는 한다. 하지만, 그런 방법을 사용하면 최대 연산 횟수가 100,000,000이 되어 시간 초과가 발생한다. 따라서, 다른 방식으로 접근해야 문제를 성공적으로 해결할 수 있다.

[3, 0, 6, 1, 5]

위와 같이 주어졌다고 생각해보자. 이를 정렬하면 다음과 같다.

[0, 1, 3, 5, 6]

오름차순으로 정렬되었기 때문에 특정 인덱스의 인용 횟수가 x이면 그 뒤에 있는 원소들은 무조건 x회 이상 인용되었다. **따라서, 5회 이상 인용된 논문이 5개 이상인지를 검사하기 위해서는 뒤에서부터 5번째 논문이 5회 이상 인용되었는지를 확인해야 한다.** 해당 인덱스의 논문의 인용횟수가 5회 이상인 경우, 그 뒤에 있는 논문들은 무조건 인용 횟수가 5회 이상이다.

## 해결 과정

위 예시를 바탕으로 계산 순서를 다시 살펴보자.

[0, 1, 3, 5, 6]

첫 번째에 위치한 논문의 인용 횟수는 0이다. 이는 5보다 작으므로 h의 조건을 만족시키지 못 한다.

[0, 1, 3, 5, 6]

두 번째에 위치한 논문의 인용 횟수는 1이다. 이는 4보다 작으므로 h의 조건을 만족시키지 못 한다.

[0, 1, 3, 5, 6]

세 번째에 위치한 논문의 인용 횟수는 3이다. 이는 3 이상이므로 조건이 만족된다. 따라서, 해당 배열의 h는 3이 된다.

### 기본 세팅

위 과정을 진행하기 위해서는 전달된 배열을 정렬해야 한다.

```java
Arrays.sort(citations);
```

### 차례대로 원소 검사

h의 최댓값부터 하나씩 줄여가면서 조건을 만족하는지 검사한다. h의 최솟값은 1, 최댓값은 논문의 개수이므로 이들을 이용하여 검사한다.

```java
int i;
for (i = citations.length; i > 0; i--) {
    if (citations[citations.length - i] >= i)
        break;
}

return i;
```

최댓값으로 부터 하나씩 줄여나가면서 해당하는 위치의 값이 앞에서 말한 조건 만족 시 그것이 최댓값이므로 반복을 종료하고 반환한다.

### 전체 코드

```java
import java.util.Arrays;

class Solution {
    public int solution(int[] citations) {
        Arrays.sort(citations);
        int i;
        for (i = citations.length; i > 0; i--) {
            if (citations[citations.length - i] >= i)
                break;
        }

        return i;
    }
}
```
