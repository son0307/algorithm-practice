## 문제 요약

- 고속도로를 이동하는 차량의 경로가 주어졌을 때, 이 경로들을 모두 커버하기 위해 필요한 카메라의 최소 개수를 반환하는 문제
    - 차량의 대수는 1대 이상 10,000대 이하
    - 진입/진출 지점의 좌표는 -30,000 이상 30,000 이하
    - 차량의 진입/진출 지점에 카메라가 설치되어 있어도 만난 것으로 판단

## 핵심 포인트

이 문제는 최대한 많은 경로가 겹치는 구간에 카메라를 설치하는 것이 핵심이며, 이를 위해 그리디 알고리즘을 사용해야 한다.

- 우선 진입 지점 (혹은 진출 지점)을 기준으로 정렬해야 순차적으로 탐색하며 겹치는 구간을 찾을 수 있다.
- 경로를 하나씩 가져오며 공통으로 가지는 범위를 갱신해야한다. 현재 카메라가 커버할 수 있는 범위를 유지하며, 새로운 차량이 이 범위에 포함되는지 검사한다. 포함된다면 공통 범위를 새로 계산하여 갱신한다. (해당 범위에 있는 경로 중 가장 먼저 나가는 차량의 경로의 진출 지점 기준)
- 이 방식을 통해 정렬($O(N log N)$) 후 한 번의 순회($O(N)$)로 문제를 해결할 수 있다.

동작 원리는 다음과 같다.

1. 차량의 진입 지점을 기준으로 routes 배열을 오름차순으로 정렬한다.
2. 첫 번째 차량의 경로를 현재 범위 기준으로 설정하고, 카메라 대수를 1로 시작한다.
3. 두 번째 차량부터 순회하며 범위에 포함됬는지 검사한다.
    1. 경로가 현재 범위와 겹치는 경우 : 다음 차량의 진입 지점이 현재 범위의 종료 지점보다 작거나 같다면, 카메라 하나로 두 차량을 확인할 수 있다. 이때, 기준 범위의 종료 기준을 두 차량 중 더 빨리 나가는 지점으로 갱신한다.
    2. 겹치지 않는 경우 : 다음 차량의 진입 지점이 현재 범위의 종료 지점보다 크다면, 기존 카메라로는 확인할 수 없다. 따라서, 카메라 대수를 1 증가시키고 기준 범위를 해당 차량의 경로로 갱신한다.
4. 위 과정을 모든 차량에 대해 확인할 때까지 반복한다.

### 전체 코드

```java
import java.util.Arrays;

class Solution {
    public int solution(int[][] routes) {
        Arrays.sort(routes, (r1, r2) -> r1[0] - r2[0]);
        int camera = 1;
        int[] current = routes[0];

        for (int i = 1; i < routes.length; i++) {
            if (routes[i][0] <= current[1]) {
                if (routes[i][1] < current[1]) {
                    current[1] = routes[i][1];
                }
            } else {
                camera++;
                current = routes[i];
            }
        }

        return camera;
    }
}
```

위 알고리즘의 시간 복잡도는 $O(NlogN)$이다. (N은 routes에 포함된 차량의 개수).

- Arrays.sort()를 사용한 정렬 : $O(NlogN)$
- 모든 경로를 순회하며 한 번씩 검사 : $O(N)$

결과적으로 전체 성능은 정렬 알고리즘의 복잡도에 의해 결정된다.
