## 문제 요약

- **목표 :** N일 동안 잡혀있는 상담 일정을 바탕으로, 퇴사일(N+1) 전까지 얻을 수 있는 최대 수익을 구한다.
- **조건**
    1. 각 상담은 완료하는 데 걸리는 기간(Ti)와 받을 수 있는 금액(Pi)이 정해져 있다.
    2. 상담을 진행하는 동안에는 다른 상담을 할 수 없다.
    3. 상담이 끝나는 날이 퇴사일(N+1일)을 넘어가면 해당 상담은 진행할 수 없다.
- **출력 항목** : 퇴사 전까지 얻을 수 있는 최대 수익

## 핵심 포인트

- **동적 계획법 (DP)** : 복잡한 문제를 여러 개의 작은 하위 문제로 나누어 풀고, 그 결과를 배열 등에 저장하여 중복 계산을 방지하는 알고리즘 기법이다. 이를 통해 일정한 규칙이 있는 문제를 효율적으로 해결할 수 있다.
- **역방향 탐색 (Bottom-up)** : 마지막 날(N일)부터 첫째 날(1일) 쪽으로 거꾸로 탐색하며 최적의 해를 누적해 나가는 방식이다. 이 문제에서는 미래의 일정이 현재의 선택에 의해 결정되므로, 뒤에서부터 계산하는 것이 훨씬 직관적이고 구현이 깔끔해진다.
- **방정식 세우기** : DP를 이용할땐 규칙에 맞춰서 방정식을 제대로 세우는 것이 중요하다.
    - 오늘(i일) 기준, 선택할 수 있는 경우는 상담을 진행하는 경우과 진행하지 않는 경우 2가지이다. 각 경우에 따라 얻을 수 있는 이익을 계산하고 둘 중 더 큰 값을 선택하면 된다.
    - 상담을 진행하지 않는 경우 → 다음날(i+1일)의 값을 선택한다.
    - 상담을 진행하는 경우 → 상담을 통해 얻을 수 있는 수익에 다음 상담이 가능한 날짜(i + Ti)의 수익을 더한다.
    - **최종 방정식** : `DP[i] = MAX(DP[i+1], P[i] + DP[i + T[i]])`

### 동작 원리

1. **초기화** : 1일부터 N일까지의 기간(T)과 금액(P)을 배열에 저장한다. 최대 수익을 누적할 `dp` 배열을 `n+2` 크기로 생성하고, 퇴사일(`dp[n+1]`)의 수익은 0으로 둔다.
2. **역방향 순회** : `n` 일부터 1일까지 거꾸로(`i--`) 반복문을 실행한다.
3. **기간 초과 검사** : `i` 번째 날의 상담을 진행했을 때, 퇴사일(`n+1`) 이전에 끝낼 수 있는지 검사한다. 만약 기한을 넘긴다면 해당 상담은 포기하고, 다음 날(`i+1`)까지의 최대 수익을 그대로 오늘(`i`)의 수익으로 물려받는다. (`dp[i] = dp[i+1]`)
4. **최적해 선택 (상태 전이)** : 상담을 기한 내에 마칠 수 있다면 두 가지 경우 중 최댓값을 선택한다.
    - **상담을 하지 않는 경우** : 내일(`i+1`)의 최대 수익 (`dp[i+1]`)
    - **상담을 하는 경우** : 오늘의 상담 수익 + 상담이 끝난 다음 날(`i+t[i]`)의 최대 수익 (`p[i] + dp[i+t[i]]`)
5. **결과 출력** : 1일까지 계산이 완료되면, 1일 차에 누적된 값(`dp[1]`)이 전체 기간 동안 얻을 수 있는 최대 수익이 된다.

### 전체 코드

```sql
import java.util.*;
import java.io.*;

public class Main {
    public static void main(String[] args) throws IOException {
        BufferedReader br = new BufferedReader(new InputStreamReader(System.in));
        StringTokenizer st;

        int n = Integer.parseInt(br.readLine());
        int[] t = new int[n+1];
        int[] p = new int[n+1];

        for (int i = 1; i <= n; i++) {
            st = new StringTokenizer(br.readLine());
            t[i] = Integer.parseInt(st.nextToken());
            p[i] = Integer.parseInt(st.nextToken());
        }

        int[] dp = new int[n+2];
        dp[n+1] = 0;

        for (int i = n; i > 0; i--) {
            if (i + t[i] - 1 > n) {
                dp[i] = dp[i+1];
            } else {
                dp[i] = Math.max(dp[i+1], p[i] + dp[i+t[i]]);
            }
        }

        System.out.println(dp[1]);
    }
}
```

위 알고리즘의 시간 복잡도는 다음과 같다.

- 배열을 뒤에서부터 앞으로 딱 한 번만 순회 : $O(N)$
- 반복문 내부에서 수행하는 연산 : $O(1)$

최종적으로 위 알고리즘은 $O(N)$의 시간복잡도를 가진다.
